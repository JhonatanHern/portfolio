# 🔍 Web Pentesting Pipeline

An automated, Dockerized security scanning pipeline using [ProjectDiscovery](https://projectdiscovery.io/) tools.
Designed to run safely on a standard laptop (MacBook).

```
Subfinder (subdomain discovery)
    → Katana (headless JS crawling)
        → Nuclei (vulnerability scanning)
```

---

## 📁 Repository Structure

```
web-pentesting/
├── config/
│   ├── domain.txt           ← YOUR TARGET DOMAIN (edit this)
│   ├── headers.txt          ← YOUR AUTH HEADERS (edit this)
│   └── dedupe.py            ← Auto-deduplication Python script
├── targets.txt              ← Auto-generated by Subfinder (don't edit)
├── docker-compose.yml       ← Container definitions
├── scan.sh                  ← Pipeline runner
└── output/                  ← Scan results (created automatically)
    ├── targets.txt          — Discovered subdomains
    ├── urls.txt             — Raw crawled endpoints
    ├── urls_deduped.txt     — Deduplicated URL list
    ├── urls_api.txt         — Clean API endpoints to scan
    ├── urls_js.txt          — Javascript files to scan for secrets
    ├── nuclei_results.txt   — Vulnerability findings (Raw)
    └── report/              — High-Fidelity Markdown Report (Readable + Evidence)
```

## 🚀 Quick Start

### Prerequisites

- **Docker Desktop** installed and running
- **Bash** (pre-installed on macOS/Linux)
- **Python 3** (pre-installed on macOS/Linux, used for URL deduplication)

### 1. Set your target domain

Open `config/domain.txt` and add your root domain:

```
example.com
```

### 2. (Recommended) Add authentication headers

Open `config/headers.txt` and add your session credentials:

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

or:

```
Cookie: session=abc123def456
```

> [!WARNING]
> **If you skip this step**, the scan will only cover publicly accessible pages.
> The script will warn you at runtime and continue with a public-only scan.
> You'll miss private dashboards, authenticated API endpoints, and internal JS fetch calls.

### 3. Run the scan

```bash
chmod +x scan.sh
./scan.sh
```

That's it. The script handles everything — pulling Docker images, running each
tool in order, and saving all results to the `output/` directory.

## 🔧 What Each Tool Does

| Step | Tool | Purpose |
|------|------|---------|
| 1 | **Subfinder** | Discovers subdomains of your target (DNS, APIs, certificate transparency) |
| 2 | **Katana** | Crawls every subdomain with a headless browser. Excludes static assets (`.png`, `.css`) internally. Intercepts JavaScript `fetch()` and `XHR` calls to find hidden API endpoints |
| 2.5 | **Deduplicator** | Python script (`config/dedupe.py`) intelligently deduplicates identical URL parameters and splits output into `urls_js.txt` vs `urls_api.txt`. |
| 3 | **Nuclei** | Employs Automatic Scanning (`-as`) to footprint API tech and execute relevant exploits, saving massive time. Runs JS-specific secrets checks strictly on Javascript files. |

## 🛡️ Security Coverage

The system uses a wide range of Nuclei templates categorized by tags to ensure comprehensive coverage:

### 1. Information Disclosure & Exposures (`exposure`, `token`, `api`)
-   **Secret Keys**: AWS, GCP, Azure, GitHub, Slack, and other cloud provider tokens.
-   **Sensitive Panels**: Exposed admin dashboards, database interfaces, and monitoring tools.
-   **Config Files**: `.git`, `.env`, `.svn`, `web.config`, and other sensitive configurations.

### 2. Common Vulnerabilities (`cve`)
-   **CVE Catalog**: Scans for thousands of known vulnerabilities across web servers (Nginx, Apache), CMS (WordPress, Joomla), and frameworks (Laravel, Spring).
-   **Zero-day Protection**: Constantly updated templates for the latest major exploits.

### 3. Web Attack Vectors (`lfi`, `sqli`, `generic`)
-   **LFI (Local File Inclusion)**: Attempts to read internal system files like `/etc/passwd`.
-   **SQLi (SQL Injection)**: Checks for database interaction vulnerabilities in URL parameters and forms.
-   **General Exploits**: Detection of Cross-Site Scripting (XSS) and Open Redirects through generic templates.

### 4. Technology Fingerprinting (`tech`, `js`)
-   **Stack Identification**: Detailed detection of frameworks, languages, and WAFs.
-   **JavaScript Analysis**: Scans JS bundles for hidden endpoints, API keys, and insecure library versions.

## 📋 Reading Your Results

After a scan, check the `output/` folder:

```bash
# View discovered subdomains
cat output/targets.txt

# View all raw crawled URLs
cat output/urls.txt

# View unique, deduplicated API endpoints to be scanned
cat output/urls_api.txt

# View unique javascript files to be analyzed for tokens
cat output/urls_js.txt

# View vulnerability findings (Raw)
cat output/nuclei_results.txt

# View the High-Fidelity Markdown Report
# Open the matching .md file in the output/report folder
ls -lh output/report/
```

Each Nuclei finding includes the template name, severity, matched URL, and details
about the vulnerability. The files in `output/report/` go even further, showing you
the **exact HTTP Request and Response** that triggered the finding.

## 🧹 Cleanup / Reset

To wipe all scan results and start fresh:

```bash
# Remove all output files
rm -rf output/*

# Reset the auto-generated targets file
git checkout targets.txt
```

To also remove the pulled Docker images (frees disk space):

```bash
docker rmi projectdiscovery/subfinder projectdiscovery/katana projectdiscovery/nuclei
```

## ⚠️ Legal Disclaimer

**Only scan domains you own or have explicit written authorization to test.**
Unauthorized scanning is illegal in most jurisdictions. The `config/domain.txt`
file exists to ensure you are making a conscious decision about your target.
